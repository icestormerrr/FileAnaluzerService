# Анализатор текстовых файлов (клиент-серверное приложение)

## Описание
Сетевое приложение на C# для анализа текстовых файлов. Сервер принимает файлы от клиентов, сохраняет их, подсчитывает количество строк, слов и символов, сохраняет результаты в локальной файловой системе и отправляет статистику обратно клиенту. Реализовано на ASP.NET Core (сервер) и консольном приложении .NET (клиент). Поддерживает одновременную обработку нескольких клиентов.

## Запуск сервера
1. Перейдите в папку `Server`.
2. Выполните:
```bash
dotnet restore
```
```bash
dotnet run
```
> Сервер будет доступен по адресу http://localhost:5000

### Запуск клиента
1. Перейдите в папку Client.
2. Выполните:

```bash
dotnet run
```
3. Программа запросит путь к текстовому файлу. Можно также передать путь аргументом командной строки:
```bash
dotnet run "C:\path\to\file.txt"
```

### Как пользоваться
- Клиент отправляет выбранный файл на сервер.
- Сервер сохраняет файл с уникальным именем в папку Uploads, анализирует его и сохраняет отчёт в папку Results.
- Клиент получает и отображает результат:
```
Имя файла: file.txt
Строк: 15, Слов: 100, Символов: 600
```

### Запуск тестов
1. Перейдите в корневую папку решения.
2. Выполните:
```bash
dotnet test
```
Все тесты из проекта Tests будут запущены.

### Конфигурация
Сервера:
- Переменные окружения хранятся в файлах: appsettings.json, appsettings.Development.json
- Порт можно изменить в launchSettings.json

Клиента:
- ServerUrl на данный момент захардкожен и хранится в переменной ServerUrl в Program.cs
